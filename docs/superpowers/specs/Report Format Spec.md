# 📐 币种分析报告 Spec

> Agent 输入一个币种名称，输出一份结构化分析报告。
> 核心原则：**学习导向**，不是交易信号。每份报告让读者学会新概念，同时为未来的买入决策积累判断力。

---

## 报告结构（7 个区块）

### ① 一句话定位 + 身份卡片

开头用一句大白话说清楚这个币是什么。

| 字段 | 说明 | 示例 |
|------|------|------|
| 一句话定位 | 不用术语，普通人能听懂 | "高速低费的底层公链，以太坊的主要竞争者" |
| 类别 | 属于哪个赛道 | Layer 1 / Layer 2 / DeFi / Meme / AI+Web3 / GameFi / RWA / 稳定币 |
| 市值排名 | CoinGecko 排名 | #5 |
| 当前价格 | 实时价格 | $148 |
| 历史最高 (ATH) | All-Time High | $260（当前距 ATH -43%） |
| 所在链 | 这个币运行在哪条链上 | 自身就是链 / 基于以太坊 ERC-20 / 基于 Solana SPL |
| 在哪买 | **必须标注 OKX 是否可买** | ✅ OKX 现货 SOL/USDT / ❌ OKX 未上线，需通过 DEX |

> **设计意图**：5 秒内定位"这是什么、多大、我能买吗"。`在哪买` 直接对接用户实际操作路径（OKX 钱包）。

---

### ② 项目是干什么的（大白话）

- 用 **3-5 句话** 解释项目在做什么
- **必须包含一个类比**（如："如果 ETH 是 iOS，SOL 想做 Android"）
- 不使用未解释的术语。如果必须提到术语，当场用括号解释
- 说清楚：这个项目解决了什么真实问题？如果没解决真实问题（如 Meme 币），直接说明

> **设计意图**：对应 milestone 目标"能解释 token 存在的意义，而不仅仅是投机"。

---

### ③ 代币经济学（Tokenomics）

表格形式呈现，每个指标必须带"这意味着什么"列：

| 必填指标 | 说明 |
|---------|------|
| 总供应量 (Max Supply) | 有上限 vs 无上限 |
| 流通量 (Circulating Supply) | 已经在市场上交易的数量 |
| 流通率 | 流通量 / 总供应量 的百分比。低流通率 = 未来有大量币会释放 |
| 通胀/通缩模型 | 是否有 burn 机制 / staking 机制 / 增发机制 |
| 质押率 (Staking Rate) | 被锁定的比例（如适用） |
| 大户集中度 | 前 10/20 地址持有比例 |
| 近期解锁事件 | 是否有大额 VC/团队解锁临近（解锁 = 潜在抛压） |

每个指标后标注：
- 🟢 健康 / 🟡 需注意 / 🔴 危险信号

术语区块：列出本节涉及的所有术语及一行解释。

> **设计意图**：Phase 3 的核心学习目标前置。用户每研究一个币就接触一次 tokenomics 实例。

---

### ④ 近期动态 & 催化剂

分两组列出：

**🟢 利好因素**（2-3 条）
- 每条一句话描述 + 术语解释

**🔴 风险事件**（2-3 条）
- 每条一句话描述 + 术语解释

要求：
- 引用的事件需要标注时间（大约什么时候发生的）
- 如果没有近期重大事件，说"近期无重大催化剂"而不是硬编

> **设计意图**：和 Telebot 每日推送形成关联。研究完一个币后，后续推送中出现相关新闻就能读懂了。

---

### ⑤ 风险评估（交通灯系统）

| 维度 | 评级 | 说明 |
|------|------|------|
| 项目风险 | 🟢/🟡/🔴 | 项目本身的技术和运营风险 |
| 代币经济风险 | 🟢/🟡/🔴 | 供给结构、解锁压力、集中度 |
| 流动性风险 | 🟢/🟡/🔴 | 能不能随时买卖、会不会卡住 |
| 监管风险 | 🟢/🟡/🔴 | 是否有被 SEC 等监管机构盯上 |
| **新手适合度** | 🟢 适合 / 🟡 谨慎 / 🔴 不建议 | **综合判断：以当前零经验状态，是否适合碰这个币** |

综合风险：🟢/🟡/🔴 + 一句话总结

新手适合度判断标准：
- 🟢 适合：主流币、OKX 可买、流动性好、不需要链上操作
- 🟡 谨慎：中等市值、有一定波动、需要理解特定概念
- 🔴 不建议：小市值、高波动、需要链上操作、合约风险高

> **设计意图**：直接回答"我现在能碰这个币吗"，对接风控框架。

---

### ⑥ 操作参考

> ⚠️ 这不是投资建议，是帮助练习"建立买入逻辑"的模板。

| 字段 | 内容 |
|------|------|
| 买入方式 | DCA 分几次 / 一次性（对接仓位管理原则） |
| 建议单次金额 | 基于 $50 初始预算的分配建议 |
| 止损参考 | 建议止损位及对应价格（-15% ~ -20%） |
| 目标位参考 | 合理的盈利目标及对应价格 |
| 持有周期 | 短期(< 1 周) / 中期(1-3 月) / 长期(> 3 月) |

术语：
- DCA = Dollar Cost Averaging，分批买入。把钱分成几份，不同时间点买入，避免一把梭在最高点
- 止损 = 价格跌到某个点就卖出，防止亏损扩大

> **设计意图**：直接对接 Phase 2 交易日志模板（买入原因 / 买入价 / 止损位 / 预期目标）。Agent 给出参考值，用户决定是否采纳并记录理由。

### ⑦ 本次研究学到了什么

```markdown
## 🧠 学习收获

本次研究涉及的关键概念：
- [ ] 概念 A — 一句话解释
- [ ] 概念 B — 一句话解释
- [ ] 概念 C — 一句话解释

→ 新术语记录到 [[📚 术语表]]：术语1, 术语2, 术语3
```

要求：
- 列出 3-5 个本报告涉及的核心概念
- 每个概念附带一行解释
- 以 checkbox 形式呈现，用户自行勾选"我能用自己的话解释这个概念"
- 最后一行列出应该记录到术语表的新术语

> **设计意图**：闭环到术语表积累，把"被动阅读"转为"主动确认理解"。

## 输出格式要求

- 输出为 Markdown
- 文件命名：`{币种符号}.md`（如 `SOL.md`、`BTC.md`）
- 存放路径：`02_WEB3/🔍 币种研究/`
- 所有术语解释使用统一格式：`术语 = 一句话解释`
- 报告中涉及的币种赛道使用 MILESTONES.md Phase 3 的分类体系：
  - Layer 1 / Layer 2 / DeFi / Meme / AI+Web3 / GameFi / RWA / 稳定币

## 用户背景（Agent 需知道）

- Web3 零交易经验，仅有 OKX 钱包
- 目标：快速积累认知，尽快可以开始投资
- 初始预算：$50（Phase 2）
- 当前阶段：Phase 1（信息基础设施），3 天后进入 Phase 2（第一笔交易）
- 风控规则：不开合约、只做现货、单笔不超过总仓位 20%、必须有买入逻辑
